# ROADMAP — что построено и что осталось

> Принцип строительства «сначала весь фронт на мок-данных → бэкенд → крипта» (ADR 0001) сработал
> и остаётся правилом для новых фич: они входят через `DataProvider` (CLAUDE.md §3), не трогая
> экраны. Подробная история фаз с чек-листами — в git-истории этого файла; техническое состояние
> построенного — `docs/yellow-paper.md`; журнал миграции на канистры — `docs/migration-plan.md`.

---

## Построено (по фазам, кратко)

- **Фаза 0–1 — UI/UX на моке.** Все экраны, состояния (empty/loading/error), потоки донатера/
  стримера/модератора/оператора; инварианты §4 видны в UI; доступность (фокус, reduced-motion).
- **Фаза 2 — бэкенд.** RPC-мост `/api/v1/rpc`, детерминированный движок репутации (общий с моком),
  модерационный конвейер с подключаемым авто-слоем (OpenAI — по ключу), дедуп по content_hash,
  очередь HELD→SHOWN/HIDDEN, баны/блок-листы/инцидент-лог.
- **Фаза 3 — крипта (devnet).** Кошелёк + SIWS, донат-tx 97/3 + memo (донор → стример напрямую),
  трастлесс-приём (сервер сам читает цепь, finalized), ончейн-активация канала. Стек — web3.js v1
  (ADR 0004); реальные кошельки без заглушек (ADR 0005); хардненг R1–R10 (ADR 0012).
- **Фаза 4 — постоянство.** PGlite-Postgres (`.data/pg/`), фоновый индексер-сервис (донаты
  доганяются без браузера), `/dev/db`, мобильная вёрстка. ADR 0014.
- **Трастлесность (ADR 0019).** Аттестация payout подписью владельца (H1), публичный экспорт +
  независимый пересчёт (`verify-export`), пруф-якорь дайджестов в devnet-memo.
- **Игра G3a — «задание-донат» (devnet).** Некастодиальный эскроу (Anchor, клейм-модель, 97/3,
  permissionless `resolve_timeout`), ончейн-гейт показа текста (ESC-19), event-индексер клеймов,
  банковка репутации строго по ончейн-исходу. Аудиты ESC-1…19 — `docs/audit-map.md`.
  Оффчейн-бэкенд игры (машина состояний, таймеры, модерация текста задания) построен в ходе G3.
- **Миграция на канистры ICP, M-1…M2 (ADR 0021).** Канистра-наблюдатель пересобирает журнал из
  цепочки (golden-паритет Rust↔TS); режим `icp` — канон чтения репутации из канистры;
  governance-параметры споров — только подписью владельца с таймлоком; **споры chain-задач
  целиком в канистре** (голоса-подписи, открытое табло, вердикт и ончейн-резолв тресхолд-ключом;
  эскроу-RESOLVER = адрес канистры, ручной резолвер удалён). Экономика — по решениям владельца:
  без депозитов/залогов, кворум фикс от стримера (дефолт 1), табло открыто (yellow-paper
  §18.5-8b/8c — **не предлагать заново**). Светофоры: `npm run golden && (cd canister && cargo
  test)`, `verify-export --canister`, `dispute-smoke`.

## Осталось (в порядке очереди)

### M3 — фронт на asset-канистре (следующая, по команде владельца)

- [ ] SPA-конверсия (Vite+React Router vs `next` static export — решить по фактическому объёму боли)
- [ ] Asset-канистра + наш домен, certified assets; кожа-API остаётся на сервере
- [ ] «Сторож показа» ESC-19 в канистре: реестр «эскроу принят ⇒ текст с коммитментом обязан быть
      публичным» (трастлесс-доказательство цензуры; решение 2026-07-05 — `docs/migration-plan.md` §5)

### M4 — governance

- [ ] Контроллеры core-канистры → мультисиг; политика апгрейдов с таймлоком, процедура публична

### M5 — mainnet-гейты (всё, что осознанно отложено)

- [ ] Мейннет-хвост M0: mainnet ICP + циклы (~$25), транспорт SOL RPC canister, durable nonce,
      certified data на экспорт, 7-дневная зелёная сверка
- [ ] Эскроу-редеплой с пинами: mint→USDC (ESC-7), прод-окна + `FAST_TEST_WINDOWS=false` (ESC-9/§18.2)
- [ ] **Upgrade authority эскроу-программы** (`G1vJ…uz14`, один ключ): мультисиг + таймлок,
      в идеале сжечь после аудита
- [ ] Полный аудит (эскроу-редеплой + core-канистра, бюджет $10–50k), нагрузочная репетиция голосов
- [ ] **Юр-мастер-переменная:** юрисдикция инкорпорации + допуск США; живая юридическая
      консультация (`docs/legal-and-risk.md` §7)
- [ ] Конверты (commit-reveal) для крупных споров ≥$50 — мейннет-план (§18.5-8b)
- [ ] Модерация: продовый ключ + fail-closed политика (MOD-2), рейт-лимиты на краю (MOD-3/B5)

### Вне очереди (по мере надобности)

- [ ] Возврат живого оверлея/алертов для OBS (SSE удалён при ADR 0005; отдельный шаг стримерского
      тулинга — yellow-paper §18.5-1)
- [ ] Прод-хвосты сервера: SIWS-сессии из файла в БД, инкрементальный `saveStore`, индексер
      отдельным воркером в serverless (ROADMAP-остаток Фазы 4)
- [ ] Калибровки на тестнете: анти-флуд/активация, кворум споров, retention карантина
- [ ] M6 (опционально): перенос нетекстовой кожи в канистру (`docs/canister-architecture.md` §4.1)

## Сквозные правила

- Новая фаза/фича заканчивается обновлением этого файла и, если было архитектурное решение, —
  новым ADR в `decisions/`.
