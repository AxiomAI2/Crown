# ROADMAP — дорожная карта строительства

Принцип: **сначала весь фронтенд на мок-данных, потом бэкенд, потом крипта.** Каждая фаза имеет
критерии выхода. Не переходи к следующей фазе, пока текущая не закрыта. Отмечай выполненное прямо
здесь (`- [x]`).

Почему такой порядок:
- **UI/UX дешевле всего менять, когда за ним ничего нет.** Прокликиваем и вылизываем продукт
  целиком, прежде чем заплатить сложностью сети и инфраструктуры.
- **Фронт не одноразовый**, потому что пишется против `DataProvider` (см. `CLAUDE.md` §3). Бэкенд и
  крипта подключаются как реализации интерфейса, экраны не трогаются.

## Фаза 4 — Постоянство и надёжность

**Цель:** данные переживают перезапуск без внешней инфраструктуры; ончейн-донаты доганяются независимо
от того, открыт ли браузер донатера.

- [x] Локальный Postgres через **PGlite** (встроенный WASM, каталог `.data/pg/`; без установки/Docker/sudo)
- [x] Реляционное отображение домена ↔ таблицы (`src/server/db.ts`, `store-db.ts`) по `backend/schema.sql`;
      деньги в `numeric(20,0)`, журнал `ledger_events` — источник истины, число пересчитывается движком
- [x] Стор грузится из БД на старте; пустая БД → одноразовый перенос прежнего JSON-снимка (`.data/store.json`)
- [x] Фоновый индексер как сервис (`src/server/indexer-service.ts`) — стартует с сервером в chain-режиме,
      следит за трежери-ATA, курсор в `meta`; RPC из `NEXT_PUBLIC_DEVNET_RPC` (бесплатный по умолчанию)
- [x] Операторский просмотр БД на `/dev/db` (выбор таблицы, сортировка; доступ только кошельку-оператору)
- [x] Адаптивная (мобильная) вёрстка по всем экранам

**Решения:** ADR 0013 (файловый стенд-ин) → **ADR 0014** (PGlite Postgres).

**Трастлесность (2026-07-02, ADR 0019):** три улучшения без нового контракта и новых операторских прав —
**(1)** payout-адрес канала закреплён ed25519-подписью владельца (H1 закрыт: клиент донора и ingest
проверяют подпись, сервер не источник истины; легаси-канал подписывает адрес кнопкой в студии);
**(2)** публичный экспорт `GET /api/v1/export/channel/[handle]` + `/api/v1/export/anchor` и независимый
пересчёт `scripts/verify-export.ts` (§4.4 — кнопка, не декларация); **(3)** пруф-якорь: дайджесты
журнала/конфигов/операторского лога периодически публикуются memo-tx (env `ANCHOR_SIGNER_KEYPAIR`) —
переписать прошлое незаметно нельзя. Раскладка по слоям доверия — `docs/trust-layers.md`.

**Остаток на прод:** SIWS-сессии всё ещё файловые (ADR 0013) — перенести в БД; `saveStore` переписывает
таблицы целиком — сделать инкрементальным; в serverless-проде индексер вынести в отдельный воркер/крон.

**Критерии выхода:** перезапуск сервера сохраняет каналы/репутацию/донаты; донат, сделанный при закрытом
браузере, доганяется индексером; независимый пересчёт из журнала совпадает с показанным числом (§4.4).

## Сквозные нерешённые вопросы (тянуть через все фазы)

- **Мастер-переменная:** юрисдикция инкорпорации + допуск пользователей из США. Влияет на геофенс и
  на то, что вообще можно строить. Решается до мейннета.
- **Upgrade authority эскроу-программы** (кошелёк `G1vJ…uz14`): один ключ может заменить код программы →
  ончейн-гарантии эскроу условны на его честность. На devnet — осознанное удобство итераций; **до
  мейннета:** мультисиг + таймлок (окно, за которое пользователи успевают выйти через permissionless
  `resolve_timeout`/`claim`), в идеале сжечь ключ после G3b-аудита.
- Калибровка анти-флуда: размер сбора активации, скорость карантина/бана, rate-limit, retention.
- Калибровка репутации: кривая (linear / bracket), decay — на тестнете.
- Политика бана: только креатор-роль или полный бан кошелька.
- Глубина ревью-восстановления для ложных срабатываний.
