# ADR 0018 — Главная (`/`) — личная домашняя база, а не каталог каналов

**Статус:** реализовано v1 — 2026-06-30 (read-only). `DataProvider.homeFeed()` + два состояния `/` +
`/discovery` (грид понижен). Остаётся уточнить (см. «Открытые вопросы»): velocity — пока прокси по свежести
createdAt; учитывается только игра `escrow-task`; отсчёт дедлайна — статичный (без посекундного тикера).
**Контекст:** сейчас `/` ([src/app/page.tsx](../src/app/page.tsx)) — это `useDiscovery()` → `ChannelBrowser`,
т.е. **грид карточек каналов** (`grid sm:grid-cols-2 md:grid-cols-3`). Грид всегда ощущался «натянутым».
Причина вскрылась через вопрос **кто физически открывает `/`**: холодный гость приходит по ссылке стримера
на КАНАЛ (`/c/[handle]`), а не на корень. Значит `/` почти всегда открывают **вернувшиеся свои**
(забукмаркали, зашли проверить). Грид проектировался под новичка, которого на этой странице нет, —
оттого и «куча рандомных карточек» не клеилась.

---

## Проблема (одной фразой)

Главная спроектирована под холодного гостя (каталог-грид), хотя её аудитория — вернувшийся участник,
у которого вопрос один: «есть ли что-то, что требует меня прямо сейчас?».

## Решение (что фиксируем)

`/` — **одна страница в двух состояниях, ни одно из которых не грид**:

1. **Есть свои циклы → дашборд открытых циклов** (позвоночник страницы). Каналы, куда ты заносил, где
   цикл ещё не закрыт: живой таск, незакрытый спор, доступный claim, грейс-окно отмены. **Сортировка по
   срочности** (см. ниже). Это та же ретеншн-механика незакрытых циклов (ADR 0015), но теперь — роль
   главного экрана.
2. **Своего нет → срез живого** (полоска «прямо сейчас»): горящие таски, идущие споры. Для любопытного
   это демка-как-маркетинг — живой момент вместо питча.

**Правило:** есть твоё → показываем твоё; нет твоего → показываем что кипит. **НИКОГДА** — «вот каналы,
выбирай с кем познакомиться». Discovery-грид не удаляется, а **понижается** до второго слоя (полоска под
личным / отдельная `/discovery` для тех, кто реально хочет полистать) — discovery «на выходе/в простое»,
не «на входе».

### Уточнения (разбор инсайта против реальной механики)

- **Линза — участник/донатер.** «Каналы, куда ты заносил» — это вид участника. Домашняя база стримера —
  входящие таски, это `/studio`. Итого: `/` = дашборд участника; стример живёт в Студии (+ его участник-циклы
  на `/`, со ссылкой в Студию).
- **Порядок срочности:** (1) действие-с-дедлайном (окно спора закрывается / доступен claim / грейс-отмена
  истекает) → (2) ждём других (таск в работе, идёт голосование) → (3) улажено. Внутри уровня — по остатку времени.
- **Единица ранжирования живой полоски — КАНАЛ/СПОР, не одиночный таск.** Метрика — число РАЗНЫХ участников
  + velocity, **НЕ сумма баунти** (иначе фронтпейдж выкупается китами: $1000 от одного = выкупил главную;
  distinct-участники = заодно анти-Sybil/анти-whale, согласовано с §4.3). Но один таск = один фандер, поэтому
  «distinct-участники» дискриминирует только на агрегате (канал с многими тасками/донатерами, спор с многими
  голосующими) — ранжируем сцену, а не отдельный таск. «50×$20 интереснее, чем кит на штуку».
- **Третье (вырожденное) состояние:** своего нет И живого нет (рано в жизни проекта) → честное пустое
  («пока тихо» + один CTA), НЕ грид и НЕ фейк (конвенция §7). «Никогда не грид» ≠ «белый экран».
- **Приватность (§4.6):** живая полоска показывает сумму / velocity / число участников / канал, но **не текст
  таска** (текст доната приватен до показа) — кроме уже показанного.

## Архитектура

Только чтения → чистый `DataProvider`, без денег/контракта/инвариантов. Нужны два новых запроса:
- `myOpenCycles(address)` — кросс-канальная сводка не-RESOLVED тасков донора + доступных claim + споров, где
  он участвует. Сейчас состояние игры по-канальное — нужна агрегация поверх (данные у индексера/стора есть).
- `liveActivity()` — ранжированный срез живого (единица — канал/спор; distinct-участники + velocity).

`DiscoveryPage`-грид заменяется двухсостоянийной `/`; discovery демотируется (полоска / `/discovery`).

## Последствия

- Главная перестаёт быть каталогом; ретеншн-механика циклов получает роль фронтпейджа.
- Discovery не исчезает, но больше не «вход» — меньше «выбери из рандома», больше «вот что требует тебя».
- Нужны 2 read-агрегации; остальное — перекомпоновка существующих данных. Денежный контур не затрагивается.

## Открытые вопросы (решить при реализации)

- Точные пороги «горящего» (сколько времени до дедлайна = «горит») и окно velocity (события за N часов).
- Нужна ли отдельная `/discovery` или достаточно полоски «показать ещё».
- Поведение для смешанной роли (стример-он-же-участник) на `/`.
