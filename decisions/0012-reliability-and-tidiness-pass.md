# ADR 0012 — Проход по надёжности и аккуратности (R1–R10) + чистка мёртвого кода

- **Статус:** принято
- **Дата:** 2026-06-20
- **Контекст:** независимое ревью кода (надёжность / простота / аудитопригодность) после ончейн-активации.
  Репозиторий уже чист (strict TS, нет `any`, нет утёкших секретов, нет TODO/мусора), поэтому правки
  точечные. Нумерация **R1–R10** — отдельное пространство имён этого прохода, чтобы НЕ путать с C/H/M/L
  из аудита «как украсть на mainnet» (ADR 0008–0011). Карта всего — `docs/audit-map.md`.

## Решения (надёжность)

### R1 — очки репутации считаются целочисленно (детерминизм §4.4)
`pointsForAmount` считал `Math.round(fromMicro(micro) * 100)` — через `float`. На суммах, где `Number(micro)`
превышает 2^53, результат расходится с точным значением, и независимый пересчёт (инвариант §4.4) не сходится.
Теперь очки = `(micro + полшага) / 10_000` в **bigint**, `Number()` только на финальном целом.
`src/lib/reputation.ts`.

### R2 — добросовестный разбор ног в индексере
`extractDonation` брал первую ногу-нетто и первую ногу-комиссию, игнорируя возможные лишние ноги того же
mint (амаунт собирался из подмножества; чужая нога могла стать «нетто»). Теперь требуем **ровно две** ноги
mint (нетто + комиссия) для доната и **ровно одну** (в трежери) для активации; иное → не наша tx → `null`.
Это не было вектором кражи (downstream-проверка payout-ATA отбивала), но делает трастлесс-чтение честным.
`src/lib/chain/indexer.ts`.

### R3 — валидация `args` как массива на границе RPC
`fn.apply(store, body.args)` с не-массивом бросал сырой `TypeError` (утекал клиенту) либо молча запускал
метод с нулём аргументов. Теперь не-массив `args` → `BAD_ARGS` сразу после декода, `undefined` → `[]`.
`src/app/api/v1/rpc/route.ts`.

### R4 — не утекать тексты внутренних ошибок
Каждая ветка отдавала клиенту `err.message` любой пойманной ошибки (тексты web3.js/PublicKey/сбоев RPC).
Теперь общий `caughtError`: клиенту — текст ТОЛЬКО доменного `DataError`, остальное → общий текст, детали в
серверный лог. `src/app/api/v1/rpc/route.ts`.

### R5 — лимит длины текста доната на ingest
`ingest` принимал текст любой длины, если совпал хэш memo (донор мог подписать memo для мегабайтного
текста → хранение/модерация/SSE). Теперь длина проверяется против `messageMaxLen` канала; длиннее →
текст игнорируется (деньги/репутация не зависят, §4.7). `src/server/ingest.ts`.

### R6 — граница кэша дедупа модерации
`modCache` рос без предела (чистился только `__reset`). Добавлен `MOD_CACHE_CAP` с вытеснением старейших
(стенд-ин под Postgres). `src/lib/data/mock-provider.ts`.

### R7 — поздняя привязка текста возвращает успех, а не `null`
В `recordDonationFromChain` успешная поздняя привязка текста к уже принятому донату могла вернуть `null`
(мёртвая ветка `if (standing)`), и `ingest` сообщал клиенту провал при фактическом успехе. Теперь возврат
безусловный. `src/lib/data/mock-provider.ts`.

### R8 — `gate` пропускает работу при простое сервера
На сервере (`latencyScale=0`, `failMode` выкл) `gate` всё равно крутил хэш-цикл по имени метода. Ранний
выход. `src/lib/data/mock-provider.ts`.

### R9 — явная авторизация `addChannelBlock`
Было `const byModerator = requireChannelManager(...) && requireSession()` — право проверялось как побочный
эффект `&&`, значение бралось из второго вызова (одно рефакторинг-движение от поломки). Разнесено на два
явных оператора. `src/lib/data/mock-provider.ts`.

### R10 — dev-вход по адресу заглушён и в chain-режиме
Вход по голому `address` без подписи (dev-тулинг) отключался только по `NODE_ENV !== production`. На staging
в chain-режиме без `NODE_ENV=production` любой мог выдать себя за владельца/оператора при работающих
деньгах. Теперь fallback жив только при `!IS_PROD && !CHAIN_MODE`. Расширение защиты C1 (ADR 0008).
`src/app/api/v1/rpc/route.ts`.

## Решения (аккуратность / аудитопригодность)

- **Удалён мёртвый код** (grep-доказано, 0 ссылок): `components/layout/public-page.tsx`,
  `components/page-stub.tsx` (заглушки Фазы 0); хуки `useConnect`/`useDisconnect`/`useIncidentLog`; тип
  `Identity`; конст `SEED`; фабрика-хелпер `ErrNotImplemented`.
- **Переименование**: `MOCK_HARD_LIST`/`MOCK_FLAG_LIST` → `DEFAULT_*` (после удаления фикстур это
  реальный дефолт-модератор, а не мок). `src/lib/data/moderation.ts`.
- **DRY**: `IS_PROD` — единый источник в `src/lib/chain/addresses.ts`, `@/server/runtime` его реэкспортирует
  (был дубль формулы в двух местах).
- **Честная фабрика**: `createDataProvider` для `chain` больше не бросает «не реализовано» (вводило аудитора
  в заблуждение — chain реализован), а объясняет, что включается через `app/providers.tsx` (ADR 0004).
- **Стейл-копирайт**: плейсхолдер `ch-lumi` (удалённая фикстура) → `ch-…` в `ops/page.tsx`.
- **Карта аудита**: добавлен `docs/audit-map.md` — инвариант §4 → код, находки → ADR → статус, границы
  доверия, dev-поверхность.

## Проверено
- `tsc` чист; `next lint` без предупреждений; продакшн-`build` зелёный (Solana по-прежнему вне bundle
  mock/api).
- `scripts/chain-verify.ts` зелёный, включая новые регрессы R2 (лишняя нога доната → `null`) и проверку
  целочисленных очков R1.

## Открытое (не закрыто этим проходом)
- **H1** (payout диктуется сервером), **L3** (токен в `localStorage`), **M3** (нет фонового индексера) — из
  ADR 0011, остаются. **H2** (сбор активации) — закрыт ончейн-активацией (предыдущий коммит).
