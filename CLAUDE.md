## 1. Что мы строим (в одном абзаце)

Репутационная платформа для стримеров. Зритель донатит стримеру в USDC на Solana и за это
накапливает **локальную репутацию** внутри комьюнити именно этого стримера (у каждого стримера —
своя, глобального рейтинга нет). Эта репутация — непродаваемое, непередаваемое вычисляемое
число — и есть продукт. Ядро: идентичность, донаты, локальная репутация, статус (тиры/бейджи/
лидерборды), модерация текста, баны. Поверх ядра построена первая мини-игра «задание-донат»
(некастодиальный эскроу на devnet, G3a) со спорами через канистру ICP (M2). **Источник истины по
механике — код и `docs/yellow-paper.md`** (фазовые спеки удалены при чистке доков 2026-07-05).

## 3. Архитектурный замок: `DataProvider` (не нарушай)

Фронтенд **никогда** не вызывает `fetch`, RPC или Solana напрямую. Весь доступ к данным идёт через
один типизированный интерфейс `DataProvider` (контракт — `src/lib/data/provider.ts`,
описание — yellow-paper §11).

```
UI компоненты  ──►  useData() хук  ──►  DataProvider (интерфейс)
                                          ├─ MockDataProvider   всё в браузере/сторе
                                          ├─ ApiDataProvider    сервер, деньги симулируются
                                          ├─ ChainDataProvider  кошелёк + devnet (основной)
                                          └─ IcpDataProvider    + канон репутации и споры
                                                                  из канистры ICP (M1/M2)
```

Реализация выбирается через ENV-флаг (`NEXT_PUBLIC_DATA_SOURCE=mock|api|chain|icp`). Новый
источник данных = новая реализация интерфейса, **не трогая ни один экран**. Если ты ловишь себя
на том, что зовёшь сеть из компонента — стоп, это нарушение контракта.

## 4. Золотые инварианты (нарушение = баг, не фича)

Эти свойства защищают проект юридически и экономически (`docs/legal-and-risk.md`). Любой код,
который их ломает, неверен, даже если «так проще».

1. **Некастодиальность.** Оператор никогда не держит деньги доната и не имеет ключа, способного
   двигать чужие средства. Донат идёт донор → стример напрямую (минус комиссия). Эскроу в ядре нет.
2. **Деньги финальны.** Нет чарджбэков, нет возвратов в ядре. Донат подтверждён → он необратим.
   Скрытие текста — это решение о публикации, а не о деньгах.
3. **Репутация непередаваема и непродаваема.** Это вычисляемое число, привязанное к адресу, а не
   токен. Никакого перевода, обмена, рынка. Это юридический замок (нет «ценной бумаги»).
4. **Репутация детерминирована и перевычислима.** Любой может скачать историю донатов + конфиг
   стримера и независимо получить ту же цифру. Сервер не «решает», он считает по открытой формуле.
5. **Репутация в ядре только растёт, и оператор её НЕ редактирует.** Ручного списания нет — число
   всегда честная свёртка журнала (§4.4). Единственная отрицательная дельта — протокольный `DISPUTE_LOST`
   (проигранный ложный спор, слой игр). Наказание нарушителя — БЛОК кошелька/канала, который обесценивает
   репутацию, не трогая само число (аудит CR-1; раньше здесь был операторский `ADMIN_VOID` — убран).
6. **Текст приватен до показа.** Текст доната по умолчанию виден только стримеру и его модераторам.
   В публичный фид / на оверлей он попадает только после явного «Показать».
7. **Деньги ≠ показ.** Репутация и деньги начисляются сразу и не зависят от судьбы текста.

## 5. Стек (как есть в коде; решения — ADR `decisions/`)

- **Frontend:** Next.js (App Router) + TypeScript + Tailwind; компоненты — своя мини-библиотека
  поверх Radix primitives; состояние — TanStack Query поверх `DataProvider`. Язык дизайна —
  `docs/design-system.md`.
- **Backend:** Next route handlers (RPC-мост `/api/v1/rpc`), PGlite-Postgres (`.data/pg/`),
  детерминированный движок репутации как чистая функция (`src/lib/reputation.ts`).
- **Crypto:** `@solana/wallet-adapter-react` + SIWS; транзакции — **web3.js v1** (ADR 0004,
  совместимость wallet-adapter; НЕ kit/gill); эскроу — Anchor-программа (`anchor/`).
- **Канистры ICP:** Rust (`canister/`), dfx 0.32; паритет с TS-логикой держат golden-тесты.

Версии пакетов и цены внешних API меняются — **сверяй перед установкой** (`product-self-knowledge`
skill для всего, что касается Anthropic; web-поиск для Solana/модерации).

## 7. Конвенции

- **Язык:** документы и UI-копирайт — русский; код, идентификаторы, имена файлов/папок, схемы, типы,
  коммиты — английский. Не смешивай.
- **Деньги и адреса — всегда моноширинным шрифтом** в UI. Это структурное правило (числа = данные).
- **Имена в UI — от лица пользователя**, а не системы. Кнопка говорит, что произойдёт: «Показать»,
  «Задонатить», «Забанить канал», а не «Submit» / «Execute».
- **Никаких заглушек-обманок.** Если фича не готова — явный empty/disabled state с объяснением, а не
  фейковая кнопка.
- **Деньги — в минимальных целочисленных единицах** (USDC = 6 знаков → храним micro-USDC `bigint`).
  **Очки репутации — дробные** (1 USDC = 1 очко с копейками, ADR 0007); свёртка журнала идёт в целых
  micro-очках (детерминизм §4.4). Форматирование только на границе UI.
- **Изменил поведение кода — обнови `docs/yellow-paper.md`** (задетые разделы + §18). Это старший
  документ после кода; расхождение с кодом = баг документации. Новая операционная шишка → строка
  в `docs/runbook.md`.
- **Владелец проекта — не программист.** К каждой пачке правок давай два ответа: (а) что изменится
  для пользователя и как увидеть это глазами (сценарий для `docs/manual-testing.md`); (б) какая
  команда-светофор это проверяет (`npm test` / `verify-export` / …). Правка без обоих ответов —
  повод объяснить, зачем она.
- **Каждая фаза заканчивается обновлением `ROADMAP.md`** (отметить выполненные пункты) и, если было
  архитектурное решение, новым файлом в `decisions/`.

## 8. С чего начать прямо сейчас

Проект НЕ на старте: построены Фазы 0–4, эскроу-игра G3a и миграция M-1…M2 (споры на канистре).
Вход для любой задачи:

1. **`docs/yellow-paper.md`** — техническое состояние ВСЕЙ системы по коду (протоколы, константы,
   гварды, схема БД). Его **§18 — обязательное чтение ДО любого фикса**: там известные проблемы и
   ПРИНЯТЫЕ компромиссы (self-donation A2, тейкдаун без возврата CR-2, FAST_TEST_WINDOWS,
   экономика споров без депозитов и с открытым табло §18.5-8b/8c и т.п. — намеренные решения
   владельца, не баги; не «чини» и не предлагай заново).
2. **`docs/trust-layers.md`** — в каком слое (кость/мышцы/кожа) живёт твоя задача и правила слоя.
   Новый рычаг кожи на кость = новый шов: запись в его §5 и в `audit-map.md` обязательна.
3. **`docs/audit-map.md`** — какие инварианты держит код, который трогаешь.
4. **`docs/runbook.md`** — запуск и операционные грабли (`.next` после краха, HMR стора,
   finalized-задержка, стенд канистры, ключи).
5. Остатки работ — `ROADMAP.md`; миграция — шапка `docs/migration-plan.md`.
