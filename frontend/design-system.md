# Дизайн-система

> Цель Фазы 1 — вылизать UI/UX. Этот документ задаёт визуальный язык, из которого растут все экраны.
> Числа (hex, rem, ms) — стартовые токены; меняй осознанно и централизованно (CSS-переменные).

---

## 1. Направление (тезис дизайна)

Продукт живёт **рядом с прямым эфиром**: донор в моменте, стример держит панель на втором мониторе во
время трансляции, деньги реальные и необратимые. Отсюда тезис — **«пульт трансляции встречает
расчётную книгу»**. Два регистра в одном продукте, и их нельзя путать:

- **Статусный регистр** (репутация, тиры, бейджи, лидерборд) — заработанный, живой, чуть «отчеканенный».
  Энергия, но дисциплинированная.
- **Денежный регистр** (донат, разбивка комиссии, финальность) — точный, спокойный, доверительный.
  Как квитанция о расчёте. Числа моноширинные, без хайпа.

**Сигнатурный элемент** (одна запоминающаяся вещь, на неё тратим всю смелость):
**репутация как «отчеканенная печать статуса»** — standing зрителя показывается плотным, тактильным
знаком-печатью, который визуально *нельзя купить-и-продать* (это вычисленная печать, а не токен). Этот
же мотив печати — в момент финальности доната (две суммы расходятся 97/3, затем «ставится печать»).
Всё остальное вокруг — тихое и строгое.

**Чего избегаем** (это AI-дефолты, не выбор): кремовый фон + serif + терракота; near-black + один
кислотный акцент; «газетная» вёрстка с волосяными линейками. У нас dark-first, но с **двумя акцентами с
разными ролями** (статус vs деньги) — это структурный выбор, а не украшение.

## 2. Цвет

Dark-first (контекст OBS / второй монитор). Светлая тема — потом, на тех же семантических переменных.

```css
:root {
  /* Поверхности (ink, не чёрный) */
  --bg:            #0E0F13;   /* основа */
  --surface:       #16181F;   /* карточки */
  --surface-2:     #1E212B;   /* приподнятое: модалки, поповеры */
  --border:        #2A2E3A;   /* линии, разделители */
  --border-strong: #3A4050;

  /* Текст */
  --text:          #ECEEF3;
  --text-muted:    #9AA1B2;
  --text-faint:    #6B7282;

  /* Акцент СТАТУСА (заработанное, «отчеканенное») — янтарь/золото */
  --status:        #E8B04B;
  --status-dim:    #B5832F;
  --status-bg:     #2A2114;   /* фон бейджей/чипов статуса */

  /* Акцент ДЕНЕГ/ДОВЕРИЯ — резервируем строго под подтверждённое/финальное */
  --money:         #3DD6A0;
  --money-dim:     #2A9C74;
  --money-bg:      #11241D;

  /* Семантика */
  --danger:        #F0566B;   /* held/blocked/ошибка/деструктив */
  --danger-bg:     #2A1419;
  --warn:          #F0A84B;
  --info:          #6E8BFF;   /* нейтральная подсветка действий */
}
```

**Дисциплина акцентов (правило, не вкусовщина):**
- `--money` (зелёный) **только** для подтверждённых/финальных денежных состояний: «Донат прошёл»,
  «Финально», подтверждённая сумма. Никогда не как декоративный акцент. Его редкость = его смысл.
- `--status` (золото) — для тиров, бейджей, прогресса репутации, лидерборда.
- `--info` — для обычных интерактивных акцентов (ссылки, активные табы), чтобы не «жечь» два смысловых
  акцента.
- `--danger` — held/blocked/деструктив/ошибка.

Тиры по умолчанию используют свою палитру (настраивается стримером), но дефолтные цвета берём из
семейства `--status` → более насыщенные к верхним тирам:
`Новичок #9AA1B2 · Свой #7FA7C9 · Постоянный #6FC3A6 · VIP #C9A24B · Легенда #E8B04B`.

## 3. Типографика

Намеренная пара, не «Inter везде». Числа и адреса — **всегда моноширинные** (структурное правило:
числа = данные).

```css
:root {
  --font-display: "General Sans", "Hanken Grotesk", system-ui, sans-serif; /* характер, дозированно */
  --font-body:    "Geist", "Inter", system-ui, sans-serif;                  /* корпус */
  --font-mono:    "Geist Mono", "JetBrains Mono", ui-monospace, monospace;  /* числа/адреса/код */
}
```

Все шрифты свободные (Fontshare / Google / npm). Если какой-то недоступен — фолбэк уже в стеке.

Шкала (type scale, 1.250 — major third), display-роль с плотным трекингом:

| Роль | Размер | Вес | Шрифт | Использование |
|------|--------|-----|-------|---------------|
| Display XL | 3.05rem | 600 | display | hero оверлея / тир-ап |
| Display L | 2.44rem | 600 | display | заголовки экранов |
| H1 | 1.95rem | 600 | display | секции |
| H2 | 1.56rem | 600 | display | подсекции |
| H3 | 1.25rem | 600 | body | карточки |
| Body | 1.0rem | 400 | body | основной текст |
| Small | 0.875rem | 400 | body | мета |
| Caption | 0.75rem | 500 | body | лейблы, eyebrow (uppercase, tracking +0.06em) |
| **Mono / Amount** | по контексту | 500 | **mono** | **все суммы, адреса, очки, tx** |

Суммы крупно — `tabular-nums`, чтобы цифры не «прыгали». Адреса всегда усечённо `7xKp…3fQa`, по клику —
копирование.

## 4. Пространство, ритм, форма

```css
:root {
  --space-1: 4px;  --space-2: 8px;  --space-3: 12px; --space-4: 16px;
  --space-5: 24px; --space-6: 32px; --space-7: 48px; --space-8: 64px;

  --radius-sm: 6px;   --radius:   10px;  --radius-lg: 14px;  --radius-pill: 999px;
  --maxw:      1200px;   /* контентный контейнер */
}
```

- Сетка дашборда: 12 колонок, желоб `--space-5`. Контент-страницы — одна колонка `--maxw`.
- Карточки — `--surface`, бордер `--border`, радиус `--radius`. Приподнятые (модалки/поповеры) — `--surface-2`.
- Тени минимальны (dark-тема): глубину даём бордером и фоном, не размытой тенью. Допустима тонкая
  тень только под модалками/оверлеем.
- **Печать статуса** (сигнатура): бейдж тира — не плоский чип, а «отчеканенный» — лёгкий внутренний
  бевел через двойной бордер/инсет-тень в `--status`, моноширинное число очков внутри. Это единственное
  место, где позволяем себе «дорогую» деталь.

## 5. Движение

Сдержанно и осмысленно. Лишняя анимация = ощущение «сгенерировано AI».

```css
:root {
  --ease: cubic-bezier(0.2, 0.7, 0.2, 1);
  --dur-fast: 120ms; --dur: 200ms; --dur-slow: 320ms;
}
```

- **Бюджет смелости — два оркестрованных момента**, остальное мгновенно и тихо:
  1. **Финальность доната** — две суммы расходятся 97/3 (`--dur-slow`), затем «печать» (короткий
     scale+settle), сумма переключается в `--money`. Это и есть «деньги ушли, необратимо».
  2. **Тир-ап** — печать статуса прорисовывается/чеканится один раз при переходе тира.
- Hover/focus — `--dur-fast`, только изменение бордера/фона, без «прыжков».
- **Всегда** уважать `prefers-reduced-motion`: оба оркестрованных момента деградируют до мгновенной
  смены состояния.

## 6. Качество-флор (без объявлений)

- Адаптив до мобильного (донор часто с телефона; дашборд/оверлей — десктоп).
- Видимый фокус с клавиатуры на всех интерактивных элементах.
- Контраст ≥ WCAG AA для текста; проверять `--text-muted` на `--surface`.
- Empty/loading/error/disabled — спроектированы, не «дыры» (см. `components.md` и копирайт ниже).

## 7. Копирайт (тон интерфейса)

От лица пользователя, активный залог, sentence case, без филлера. Из `frontend-design` skill +
инварианты ядра:

- Кнопка говорит, что произойдёт: «Задонатить», «Показать», «Скрыть», «Активировать канал»,
  «Забанить канал». Действие сохраняет имя через весь флоу (кнопка «Показать» → тост «Показано»).
- **Финальность — честно:** перед подписью доната — «Донат необратим. Возврата нет.» После — «Готово.
  Деньги ушли стримеру.» Никаких «возможно/обычно».
- **Никаких обещаний справедливости** (см. `docs/legal-and-risk.md` §6): рамим как статус/комьюнити, не
  «выигрыш/гарантия».
- Ошибки не извиняются и не туманят: что случилось + как починить. Пустой экран — приглашение к действию.
- Адреса/суммы — данные, не проза: моно, копируемы.

## 8. Реализация (Фаза 0)

- Токены — в `:root` (CSS-переменные) + проброс в `tailwind.config` через `theme.extend` (цвета,
  spacing, fontFamily, radius), чтобы были доступны и как утилиты, и как переменные.
- Шрифты — `next/font` (локально или Fontshare/Google), с `display: swap`.
- Темизация семантическая: компоненты ссылаются на `--text`, `--surface`, `--status`, `--money` и т.д.,
  **не** на сырые hex. Светлая тема позже = переопределение переменных, без правки компонентов.
- Собери `/dev/kitchen-sink` со всеми токенами и компонентами для визуального ревью.
