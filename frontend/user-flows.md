# Пользовательские потоки

Сквозные потоки, которые должны проходиться насквозь на мок-данных к концу Фазы 1. Каждый — с
ветвлениями и инвариантами. Формат: шаги + что проверяем.

---

## Поток 1 — Донат с текстом (счастливый путь)

**Актор:** донатер. **Экран:** `/c/[handle]`.

```
1. Зритель открывает канал. Подключает кошелёк (если ещё нет) → Sign-In (gasless).
2. Видит "Моё standing": Новичок, 0 очков (если первый раз).
3. В DonateWidget вводит сумму (напр. $10). FeeSplit пересчитывается: Стримеру $9.70 · Платформе $0.30.
4. Включает "Добавить сообщение", пишет текст. Счётчик до лимита; локальный wordlist мягко
   предупреждает при ругательстве (не блокирует — это вкус стримера).
5. "Задонатить" → DonateFlow:
   а) Подтверждение: суммы + "Донат необратим. Возврата нет." → "Подтвердить и подписать".
   б) Подпись (Фаза 1: имитация; Фаза 3: кошелёк).
   в) FinalityMoment: 97/3 расходятся → печать → "Готово. Деньги ушли стримеру." (--money).
   г) Судьба текста: "Сообщение на модерации у стримера (HELD). Деньги и standing уже зачтены."
6. "Моё standing" обновляется СРАЗУ: очки +1000 (по дефолтной linear), возможно тир-ап (Поток 4).
```

**Проверяем:** деньги/репутация начисляются сразу и **не зависят** от модерации текста; финальность
показана честно; standing вырос немедленно; текст ушёл в HELD, а не сразу в ленту.

## Поток 2 — Модерация: показать / скрыть

**Актор:** стример/модератор. **Экран:** `/studio/queue`.

```
1. В очереди появляется HELD-сообщение из Потока 1, с авто-вердиктом (CLEAR/FLAG) и языком.
2а. "Показать" → состояние SHOWN:
     - сообщение появляется в ленте канала и на оверлее (Поток 3);
     - тост "Показано".
2б. "Скрыть" → состояние HIDDEN:
     - текст не публикуется нигде;
     - деньги и standing донора ОСТАЮТСЯ (микротекст под действием это напоминает);
     - тост "Скрыто".
3. Если авто-вердикт = HARD_BLOCK: сообщение в карантине, "Показать" недоступно, путь эскалации в T&S.
```

**Проверяем:** show/hide меняет только судьбу текста; HIDDEN не трогает деньги/репутацию; HARD_BLOCK
нельзя показать; дедуп повторного контента; решение из кэша для повторов.

## Поток 3 — Оверлей (OBS)

**Актор:** зритель трансляции (пассивно). **Экран:** `/c/[handle]/overlay`.

```
1. Стример захватил оверлей в OBS (прозрачный фон).
2. Когда сообщение становится SHOWN и сумма ≥ минимума показа — появляется алерт:
   донор + бейдж тира + сумма (моно) + текст (+ опц. TTS).
3. reduced-motion: алерт появляется без анимации входа.
```

**Проверяем:** на оверлей попадают ТОЛЬКО SHOWN; HELD/HIDDEN/QUARANTINED — никогда; hard-block —
никогда; уважается минимальная сумма показа.

---

## Поток 4 — Накопление и переход тира

**Актор:** донатер (с течением донатов).

```
1. После очередного доната очки пересекают порог следующего тира.
2. StandingSeal проигрывает TierUp-момент (оркестрованный, reduced-motion-aware).
3. Бейдж/цвет ника обновляются везде (лента, лидерборд, оверлей).
4. Разблокированные перки тира становятся видимыми в "Моё standing" / TierLadder.
```

**Проверяем:** тиры — чистая презентация текущих очков; смена косметики стримером применяется сразу;
переход тира — заметный, но не назойливый момент.

---

## Поток 5 — Базовый канал отклоняет донат-с-текстом

**Актор:** донатер на `BASIC`-канале.

```
1. Зритель открывает BASIC-канал. DonateWidget показывает: обычный донат доступен,
   "Добавить сообщение" — disabled с пояснением "Этот канал ещё не активирован".
2. Обычный донат (без текста) проходит нормально (деньги + standing).
3. (Стример видит на дашборде призыв активировать → Поток 6.)
```

**Проверяем:** разделение обычного минимума и минимума-с-текстом; BASIC не индексируется в Discovery;
понятное объяснение, а не «серое поле».

---

## Поток 6 — Активация канала

**Актор:** стример. **Экран:** `/studio/activation`.

```
1. BASIC-канал. Карточка активации: что разблокирует + сумма сбора (~$2) + "невозвратно".
2. "Активировать канал" → подпись/оплата (Фаза 1: имитация).
3. Статус BASIC → ACTIVE. Разблокируются: донаты-с-текстом, публичная индексация, оверлей-фичи.
4. Канал появляется в Discovery.
```

**Проверяем:** сбор тарифицирует поверхность, не существование канала; после бана возврат = новый
кошелёк + снова сбор (объяснено).

---

## Поток 7 — Флуд и лестница наказаний

**Актор:** стример → оператор. **Экраны:** `/studio/queue`, `/studio/blocklist`, `/ops`.

```
1. Один источник шлёт много донатов-с-текстом (FloodIndicator подсвечивает; дедуп схлопывает повторы).
2. Стример: "Заблокировать кошелёк на канале" → канальный блок (не платформенный).
3. Если это нелегальщина/системный абьюз → эскалация в /ops:
   лестница: скрыть → канальный блок → SUSPEND → бан креатор-роли → полный бан кошелька →
   ADMIN_VOID → (для CSAM) карантин + preservation + репорт в NCMEC.
4. SUSPENDED (авто, до ревью) ≠ BANNED (подтверждено); есть путь ревью/восстановления.
```

**Проверяем:** канальный блок ≠ платформенный бан; ADMIN_VOID — единственное списание репутации;
деструктив с подтверждением и записью в инцидент-лог; путь восстановления для ложных срабатываний.

## Поток 8 — Подключение и сессия

**Актор:** любой. **Экран:** `/connect`.

```
1. "Подключить кошелёк" → выбор кошелька (Фаза 1: мок-список; Фаза 3: реальные).
2. Sign-In: подпись сообщения (без газа, без пароля). Кошелёк = аккаунт.
3. Сессия активна; в шапке — адрес (усечённо) + бейдж тира на текущем канале.
4. Отключение → возврат к публичному виду.
```

**Проверяем:** мультикошельки не агрегируются; адрес-онли по умолчанию (профиль не требуется);
ошибки (отклонено/нет кошелька) показаны понятно.

---

## Поток 9 — Создание канала (предшествует Потоку 6)

**Актор:** будущий стример. **Экран:** `/studio/create`.

```
1. Кошелёк без канала открывает /studio → редирект на /studio/create.
2. Вводит handle (проверка уникальности/формата) и payoutAddress (по умолч. = логин-адрес).
3. "Создать канал" (createChannel) → создаётся BASIC-канал; редирект на /studio.
4. На дашборде BASIC — призыв активировать (→ Поток 6).
```

**Проверяем:** один канал на кошелёк (повторное создание недоступно — ссылка на /studio); payout-адрес
может отличаться от логин-адреса; BASIC создаётся бесплатно и не индексируется в Discovery до активации.
