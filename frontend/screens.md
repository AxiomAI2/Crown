# Экраны

> ⚠️ **Статус (2026-07-02):** частично устарела. Реализованная карта экранов — `docs/yellow-paper.md`
> §14. Здесь неактуально: Overlay view (SSE-оверлей удалён при ADR 0005, в коде отсутствует),
> настраиваемая репутационная формула (курс фиксирован, ADR 0007), '/studio/create' (создание канала —
> инлайн в обзоре студии).

Экран за экраном: задача, содержимое, состояния, инварианты. Состояния обязательны для каждого:
**empty / loading / error / disabled / success**. Где не указано иное — данные через `DataProvider`.

Легенда: 🟢 публичный · 🟠 стример · 🔴 оператор · 💰 трогает деньги · 🛡 трогает модерацию/инварианты.

## 🟢 Главная / Home — `/` (ADR 0018)

**Задача:** ответить вернувшемуся участнику на один вопрос — «есть ли что-то, что требует меня прямо
сейчас?». `/` почти всегда открывают свои (холодный гость приходит по ссылке стримера на `/c/[handle]`),
поэтому это **личная база, а не каталог**. **Никогда не грид «выбери канал».**

**Личная база = ПРОФИЛЬ (ADR 0018).** Залогинен → `/` показывает ТВОЙ профиль (`DonorProfile editable`),
совмещённый с главной: сверху секция **«Требует тебя»** (открытые циклы, owner-only, `OpenCycles`), ниже —
«куда донатил» (донаты + standing по каналам + активность). Циклы (эскроу-задания донора) отсортированы по
срочности: (1) действие-с-дедлайном (`claimable`/окно спора/грейс) → (2) ждём других (в работе/голосование);
внутри — по остатку времени. Данные: `homeFeed()` (личность из сессии; §4.6 — свой текст задания).

**Гость (не залогинен) → срез живого** («Прямо сейчас»): каналы, ранг по числу РАЗНЫХ участников + velocity
(НЕ сумма баунти; анти-whale/§4.3). Текст таска не светим (§4.6). Данные: `homeFeed().live`.

**Состояния:** профиль-база (залогинен) / live (гость, что-то кипит) / empty («пока тихо» + CTA — не грид,
не фейк) / loading (skeleton) / error (повтор).

**Discovery** (грид каналов) понижен на отдельную `/discovery` — «на выходе/в простое», не вход.

## 🟢💰🛡 Страница канала (вид донатера) — `/c/[handle]`

Главный публичный экран. Слева — контент канала, справа — донат и статус. На мобиле — стек.

**Блоки:**
1. **Шапка канала** — handle, статус (LIVE-индикатор опц.), цвет/тема канала.
2. **Моё standing на этом канале** (если кошелёк подключён) — печать статуса (сигнатура из
   `design-system.md`): тир, очки (моно), прогресс до следующего тира, всего задонатил. Если не
   подключён — призыв подключить. Если подключён, но не донатил — тир «Новичок», 0 очков, «Сделай
   первый донат, чтобы начать набирать standing».
   - **Инвариант:** рядом со standing **нет** кнопок «передать/продать». Можно показать тултип
     «Standing нельзя купить или передать — он считается из твоих донатов» (это и UX, и юр.-замок).
3. **Донат-виджет** (см. поток в `user-flows.md`):
   - поле суммы (USDC, моно), быстрые пресеты;
   - валидация: сумма ≥ `max(газовый флор, минимум стримера)`; для текста — отдельный минимум;
   - переключатель «добавить сообщение» → текстовое поле (счётчик до лимита, локальный wordlist на
     вводе с мягким предупреждением);
   - **разбивка комиссии** прямо в виджете: «Стримеру 97% · Платформе 3%» с конкретными суммами (моно);
   - кнопка «Задонатить» (disabled пока невалидно).
4. **Лента показанных сообщений (SHOWN)** — донор, бейдж тира, сумма (моно), текст, время. Только
   SHOWN; HELD/HIDDEN тут не видны (инвариант приватности).
5. **Лидерборд** — табы all-time / месяц / топ-донатер месяца (видимость — рычаг стримера, может быть
   скрыт). Ранг, донор (бейдж/цвет тира), очки (моно).
6. **Тиры канала** — компактная лестница тиров с порогами и перками (что разблокирует).

**Состояния:** канал `BASIC` (донат-с-текстом отключён, объяснить почему); канал `SUSPENDED`/`BANNED`
(страница недоступна, нейтральное сообщение); кошелёк не подключён; пустой лидерборд; ошибка доната.

## 🟢💰🛡 Донат-флоу (модалка/шаги) — поверх `/c/[handle]`

Подробный поток — `user-flows.md`. Шаги-состояния:
1. **Ввод** — сумма + опц. текст + разбивка комиссии.
2. **Подтверждение** — «Стримеру $X · Платформе $Y · Итого $Z». **Предупреждение о финальности:**
   «Донат необратим. Возврата нет.» Кнопка «Подтвердить и подписать».
3. **Подпись** (Фаза 3 — кошелёк; Фаза 1 — имитация) — состояние ожидания.
4. **Финальность** (сигнатурный момент, `design-system.md` §5): суммы расходятся 97/3, «печать»,
   переключение в `--money`. «Готово. Деньги ушли стримеру.»
5. **Судьба текста** — если был текст: «Сообщение на модерации у стримера (HELD). Деньги и твоё standing
   уже зачтены — публикация текста от них не зависит.»
**Состояния ошибки:** отклонено кошельком, недостаточно средств/газа, сеть/таймаут, сумма ниже минимума.

## 🟢 Профиль зрителя / My standing — `/me`

**Задача:** свести standing зрителя по всем каналам + история донатов.
**Содержимое:** список каналов, где есть standing (печать статуса на каждый, тир, очки, всего задонатил);
история донатов (канал, сумма моно, было ли сообщение и его состояние, время, tx-ссылка в Фазе 3).
**Инвариант:** репутация **локальна** — никакого «общего рейтинга по платформе». Подчеркнуть это в UI
(«У тебя своё standing в каждом комьюнити»).
**Состояния:** не подключён; подключён без донатов (empty с призывом); loading; error.

## 🟢 Настройки лёгкого профиля — `/me/profile`

**Задача:** opt-in профиль (display_name, avatar, bio, links).
**Содержимое:** форма; явно сказано, что профиль необязателен и что включение добавляет публичную
поверхность. Превью, как ник выглядит в ленте/лидерборде.
**Состояния:** профиль выключен (адрес-онли, дефолт) / включён; валидация полей; сохранено.

## 🟠 Создание канала — инлайн в `/studio` (обзор)

> Не отдельный маршрут: если у кошелька нет канала, обзор студии показывает форму создания на своём месте.

**Задача:** превратить кошелёк в стримера: создать канал (`createChannel`). Это первый шаг до активации.
**Содержимое:** форма — `handle` (публичный slug, проверка уникальности и формата) + `payoutAddress`
(адрес для выплат, по умолчанию = логин-адрес, но можно указать другой; моно, валидация base58).
Пояснение: создаётся `BASIC`-канал — бесплатно, но без донатов-с-текстом и без публичной индексации
(их разблокирует активация — баннер в студии).
**Инвариант:** **один канал на кошелёк** (`spec.md` §2). Если у кошелька уже есть канал — вместо формы
показывается обычный обзор канала.
**Состояния:** нет кошелька (призыв подключить); форма (пусто/валидация/handle занят); создаётся;
успех → обзор канала (`BASIC`); error.

## 🟠 Дашборд стримера — `/studio`

**Задача:** обзор канала за сегодня/период.
**Содержимое:** статус канала (`BASIC`/`ACTIVE`/…); метрики (донатов, оборот моно, новые донатеры,
сообщений в очереди); последние донаты; быстрые ссылки (Очередь, Настройки, Оверлей).
Если канал `BASIC` — крупный призыв активировать (баннер активации в студии) с объяснением, что разблокируется.
**Состояния:** **нет канала у кошелька → форма создания прямо в обзоре**; `BASIC` (ограниченный дашборд +
активация); `ACTIVE` пустой (нет данных); loading; error; `SUSPENDED`/`BANNED` (баннер с объяснением и
путём ревью).

## 🟠🛡 Очередь модерации — `/studio/queue`

**Задача:** стример/модератор решает судьбу текста. Это центр ежедневной работы.
**Содержимое:** список HELD-сообщений; на каждом: донор (бейдж тира), сумма (моно), текст, **авто-вердикт**
(CLEAR / FLAG / HARD_BLOCK с пометкой), язык. Действия: **«Показать»** / **«Скрыть»**. HARD_BLOCK —
авто-карантин, показать нельзя (объяснить), путь эскалации в T&S.
**Важно (инвариант):** под каждым сообщением напоминание-микротекст «Деньги и standing донора зачтены —
решение касается только публикации текста.»
**Фичи:** фильтр по вердикту/языку; пакетные действия; дедуп (повтор того же контента схлопнут);
rate-limit/флуд-индикатор; кнопка «Заблокировать кошелёк на канале» (`/studio/blocklist`).
**Состояния:** пусто («Очередь чиста»); только флаги; HARD_BLOCK-карантин; loading; error; авто-показ
включён (объяснить, что часть проходит автоматически с задержкой, кроме hard-block).

## 🟠 Настройки канала — `/studio/settings`

**Задача:** все рычаги стримера (`core-spec.md` §7). Разбить на секции (табы/аккордеон):

1. **Тиры и пороги участия** — редактор тиров (имя, порог в очках, цвет ника, бейдж, перки). Курс
   репутации фиксирован (`1 USDC = 1 очко`, ADR 0007), не настраивается — пояснить это в секции.
   Пороги — сколько очков нужно для тира/перков/участия в мини-играх; применяются к текущим очкам.
3. **Донаты** 💰 — минимум обычного / минимум с текстом (раздельно), лимит длины сообщения,
   профанити-политика (маскировать / скрыть / в очередь).
4. **Имена** — режим: только адреса / разрешить чужие display_name (поверхность модерации).
5. **Показ текста** 🛡 — ручное одобрение (дефолт) / авто-показ-если-чисто (с задержкой; пояснить, что
   hard-block никогда не авто-показывается).
6. **Оверлей / алерты** — стиль, звук, минимальная сумма показа, TTS.
7. **Модераторы** — список кошельков + скоуп (`queue` / `queue_and_block`).

**Состояния:** загрузка конфига; валидация (например пороги тиров по возрастанию); несохранённые
изменения; сохранено (новая `config_version`); ошибка.

## 🟠💰 Активация канала — баннер во вкладках студии

> Не отдельный маршрут: контекстный баннер активации (`ChannelStatusBanner`) висит во всех вкладках студии,
> пока канал `BASIC`, и исчезает после активации.

**Задача:** одноразовый сбор (~$2 USDC) → `BASIC → ACTIVE`.
**Содержимое:** что разблокирует (донаты-с-текстом, публичная индексация, оверлей-фичи); сумма сбора
(моно); кнопка «Активировать канал»; пояснение, что сбор **невозвратный** и тарифицирует атакуемую
поверхность, а не существование канала.
**Состояния:** `BASIC` (доступно); в процессе подписи/оплаты; `ACTIVE` (уже активирован — показать дату);
ошибка оплаты. После бана — объяснить, что возврат = новый кошелёк + снова сбор.

## 🟠 Канальные блокировки — `/studio/blocklist`

**Задача:** список заблокированных на канале кошельков (не платформенно).
**Содержимое:** таблица (адрес, причина, кем, когда), действие «Разблокировать». Пояснить разницу:
канальный блок ≠ платформенный бан (тот только у оператора).
**Состояния:** пусто; список; loading; error.

## 🟢 Оверлей для OBS — `/c/[handle]/overlay`

**Задача:** публичный вид без хрома для захвата в OBS.
**Содержимое:** алерты новых SHOWN-донатов (донор, бейдж тира, сумма моно, текст, опц. TTS), уважение
минимальной суммы показа; опц. бегущий топ-донатер/лидерборд. Прозрачный фон (для chroma/overlay).
**Состояния:** ожидание (нет алертов); поток алертов; reduced-motion (без анимаций входа).
**Инвариант:** только SHOWN. HELD/HIDDEN/QUARANTINED сюда не попадают никогда; hard-block — никогда.

## 🔴🛡 Консоль оператора / T&S — `/ops`

**Задача:** платформенный уровень: то, что стример не может (`core-spec.md` §9).
**Содержимое:**
- очередь эскалаций (FLAG/HARD_BLOCK/репорты зрителей) с языковой пометкой;
- лестница наказаний: скрыть → канальный блок → `SUSPEND` → бан креатор-роли → полный бан кошелька →
  `ADMIN_VOID` → юр.-эскалация (карантин + preservation + репорт в NCMEC);
- санкционный скрининг адресов (AML), блок при попадании;
- инцидент-лог (kind, detail, resolution, preservation/reported флаги);
- разбор `SUSPENDED` (авто) vs `BANNED` (подтверждено) + путь ревью/восстановления для ложных
  срабатываний.
**Инварианты:** `ADMIN_VOID` — единственное списание репутации; деструктивные действия с подтверждением
и записью в инцидент-лог; для CSAM путь карантин+preservation+репорт обязателен.
**Состояния:** пусто; очередь; деструктивное подтверждение; loading; error.
**Примечание:** в Фазе 1 — на мок-данных; реальная логика — Фазы 2–3. Но экран и состояния
проектируем сейчас.

## Системные экраны / состояния (сквозные)

- **Connect / Sign-In** — кнопкой в шапке (модалка wallet-adapter), не отдельный маршрут: выбор кошелька,
  подпись SIWS-сообщения. Ошибки: отклонено, кошелёк не найден.
- **Забаненный/саспенднутый канал** — нейтральная страница, без деталей, с путём обращения.
- **404 / пустой handle.**
- **Глобальные:** тосты (успех/ошибка/инфо), skeleton-загрузки, баннер «несохранённые изменения»,
  диалоги подтверждения деструктива.
