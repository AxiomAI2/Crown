# Библиотека компонентов

Собственная мини-библиотека поверх Radix primitives, стилизованная по `design-system.md`. Каждый
компонент рендерится во всех релевантных состояниях на `/dev/kitchen-sink`.

Конвенции: TypeScript, props типизированы из `mock-data.md`; деньги/очки/адреса форматируются
утилитами (`spec.md` §4), не вручную; семантические CSS-переменные, не сырые hex.

---

## 1. Примитивы (Фаза 0)

| Компонент | Заметки |
|-----------|---------|
| `Button` | варианты: `primary` / `secondary` / `ghost` / `danger` / `money`. `money` — для подтверждённого денежного действия. Состояния: default/hover/focus/loading/disabled. |
| `Input`, `Textarea` | с лейблом, хелпером, ошибкой; `Textarea` — со счётчиком символов. |
| `AmountInput` 💰 | спец-инпут суммы USDC: моно, маска, пресеты, мин/макс-валидация, показывает ошибку «ниже минимума». |
| `Select`, `Switch`, `Tabs`, `Tooltip`, `Dialog`, `Popover`, `Toast` | обёртки Radix. |
| `Skeleton` | для loading-состояний (карточки, строки, печать статуса). |
| `EmptyState` | иконка + заголовок + текст + опц. действие. Копирайт «приглашение к действию». |
| `ErrorState` | что случилось + как починить + «Повторить». Не извиняется, не туманит. |
| `ConfirmDialog` | для деструктива (бан, void). Требует явного подтверждения. |
| `CopyableAddress` | `7xKp…3fQa`, клик = копировать, тост «Адрес скопирован». |
| `Amount` | отображение суммы: моно, tabular-nums, опц. вариант `money` (зелёный, для финального). |

## 2. Доменные компоненты

### Статус / репутация (статусный регистр)

| Компонент | Что показывает | Состояния |
|-----------|----------------|-----------|
| `StandingSeal` ⭐ | **Сигнатура.** «Отчеканенная печать» standing зрителя: тир, очки (моно), цвет. | по тиру; loading (skeleton); 0-очков (Новичок); анимация тир-апа |
| `TierBadge` | компактный бейдж тира рядом с ником/в ленте/лидерборде | по тиру |
| `ReputationProgress` | прогресс до следующего тира (0..1) + «осталось N очков» | есть/нет следующего тира |
| `TierLadder` | лестница тиров канала с порогами и перками | редактируемая (settings) / read-only (канал) |

## 3. Состояния-обязанность (для каждого компонента с данными)

```
loading  → Skeleton (форма совпадает с финальной, без layout-shift)
empty    → EmptyState (приглашение к действию)
error    → ErrorState (что + как починить + Повторить)
disabled → визуально явно + причина (тултип/хелпер)
success  → нормальный рендер
```

> Правило: компонент не «исчезает» в loading/empty/error — он показывает соответствующее состояние.
> Это часть того, что делает продукт понятным без спеки.
